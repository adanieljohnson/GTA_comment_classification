Overview

This notebook summarizes my progress, thoughts while building a working Ensemble method for categorizing TA comments.

 

H-Stacking Replicate Cycles of Naive Bayes

One of my ideas was to see if I could H-stack Naive Bayes by running the same classifier with different randomized datasets, then putting them side by side and assigning final categories based on most frequent classification in the set.

To test this I ran 5 Subject classifier replicates using the code in Naive_bayes_V2r_2b.rmd, with 5 different set.seed values. The exported CSV files were combined to get a set of data with up to 5 classifier assignments for the same comment.

Using multiple randomized NB training runs does not seem to help. First, it was rare for the randomized runs to assign different incorrect classes. Second, when I compared 5 independently randomized runs by eye, the incorrectly categorized items were repeatedly put into the SAME incorrect group each time. Based on this, I expect that an Ensemble method using H-stacked categorical NB is not going to reduce overall errors in classification.

Looking at the combined data DID provide other helpful insights into the overall classification procedure though. Naive Bayes does not seem to do well with comments that:

  • Contain many or nearly all punctuation symbols like:
    • : not ,
    • ?
    • ??
    • ???
  • Are whole numbers
    • 1
    • 2
    • 10
  • Are one word questions or comments
    • because?
    • “and”
  • Are sentences enclosed in punctuation
  • Are made up entirely of STOP/common words (? not as sure)

 

Overall Accuracy

I also looked at OVERALL accuracy of NB by recreating the 2-dimensional contingency table that I made when hand-coding, and comparing it side-by-side to NB classifications. From this I gained additional insights.

  • NB over-classifies comments as “copyediting of technical issues” by 8-10%, and under-classifies comments that should be marked as “specific technical issues” by almost same amount.
  • Comments that belong to groups that make up less than 1-2% of all comments seem more likely to be mis-classified.
  • Longer text comments seemed to be mis-classified more often.
    • So should I split into shorter phrases?
    • Other option is to only use the first 5-10 words.
  • NB seems to perform better for identifying the SUBJECT of the comment, and is weaker for the STRUCTURE.
    • I know structure comments are more based in sentiment and sentence structure, while subject labels are more terms-centered.
    • Do I need to incorporate a sentiment classifier or PoS tagger step?

 

Overall, 2-D Naive Bayes performed very well for comment subject. Overall, it scored the dataset within 1.5% of the hand-scored data. So it may still be useful.

 

Mixed Ensemble Method

It seems like the most reasonable approach for maximum accuracy is going to be a mixed ENSEMBLE method. My working plan is:

  1. Isolate “short-string” comments first. Look for single words, two-word phrases, more punctuation than text, etc. These represent mostly pointers and copy-editing.

  2. Look at 2-gram frequencies in each category. (Ex., “basic criteria” 2-gram identifies a Basic Criteria comment.)

  3. These are the pre-screening step prior to implementing NB.
